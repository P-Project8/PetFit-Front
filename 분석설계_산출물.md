## 1. Use Case Diagram (유스케이스 다이어그램)

PetFit 시스템에서 각 액터(사용자)가 수행할 수 있는 기능을 정리한 다이어그램.

### 액터(Actor) 정의

| 액터 | 설명 |
|------|------|
| 비회원(Guest) | 로그인하지 않은 사용자. 상품 탐색·검색·갤러리 조회 등 열람 위주 기능 사용 |
| 회원(Member) | 로그인한 사용자. 구매·AI 스타일링·반려견 관리 등 전체 기능 사용 |
| AI 시스템 | 가상 피팅 이미지를 합성하는 외부 AI 서버 |

```mermaid
flowchart LR
    Guest(["비회원<br/>Guest"])
    Member(["회원<br/>Member"])
    AI(["AI 시스템"])

    subgraph PetFit["PetFit 시스템"]
        UC1(("회원가입/이메일인증"))
        UC2(("로그인/로그아웃"))
        UC3(("상품 탐색/카테고리"))
        UC4(("상품 검색"))
        UC5(("상품 상세 조회"))
        UC6(("갤러리 피드 조회"))

        UC7(("AI 스타일링<br/>가상 피팅"))
        UC8(("반려견 프로필 관리"))
        UC9(("사이즈 추천"))
        UC10(("장바구니"))
        UC11(("찜(위시리스트)"))
        UC12(("주문/결제"))
        UC13(("주문 취소/내역"))
        UC14(("리뷰 작성/관리"))
        UC15(("갤러리 공유/좋아요/댓글"))
        UC16(("프로필 수정"))
    end

    Guest --- UC1
    Guest --- UC3
    Guest --- UC4
    Guest --- UC5
    Guest --- UC6

    Member --- UC2
```

## 2. Sequence Diagram (시퀀스 다이어그램)

핵심 기능인 **AI 스타일링(가상 피팅)** 의 동작 흐름을 시간 순서대로 표현.

### 시나리오: 회원이 반려견에게 옷을 입혀보고 결과를 갤러리에 공유

```mermaid
sequenceDiagram
    actor User as 회원
    participant FE as 프론트엔드<br/>(React)
    participant BE as 백엔드 서버
    participant AI as AI 합성 서버
    participant S3 as 파일 저장소<br/>(S3)
    participant DB as DB

    User->>FE: AI 스타일링 페이지 진입
    FE->>BE: 내 반려견 목록 조회 (GET /api/pets)
    BE->>DB: 반려견 데이터 조회
    DB-->>BE: 반려견 목록
    BE-->>FE: 반려견 목록 반환
    FE-->>User: 반려견 선택 화면 표시

    User->>FE: 반려견 사진 + 옷(상품) 선택
    FE->>BE: AI 스타일링 요청<br/>(POST /api/ai/styling)
    Note over FE,BE: 반려견 이미지, 옷 이미지,<br/>productId, petProfileId 전달

    BE->>AI: 가상 피팅 이미지 합성 요청
    AI-->>BE: 합성 결과 이미지
    BE->>S3: 결과 이미지 업로드
    S3-->>BE: 이미지 URL
    BE->>DB: 스타일링 결과 저장
    BE-->>FE: 결과 이미지 URL 반환
    FE-->>User: 합성 결과 표시 (로딩 → 결과)

    FE->>BE: 유사 상품 추천 요청<br/>(GET /api/products/filter)
    BE-->>FE: 동일 카테고리 추천 상품
    FE-->>User: 유사 상품 추천 노출

    User->>FE: 갤러리에 공유하기
    FE->>BE: 갤러리 게시글 생성<br/>(POST /api/gallery)
    BE->>DB: 게시글 저장
    BE-->>FE: 생성 완료
    FE-->>User: 갤러리 공유 완료
```

### (보조) 시나리오: JWT 토큰 자동 재발급

```mermaid
sequenceDiagram
    actor User as 회원
    participant FE as 프론트엔드
    participant BE as 백엔드 서버

    User->>FE: 보호된 기능 요청 (예: 주문)
    FE->>BE: API 요청 (만료된 accessToken)
    BE-->>FE: 401 Unauthorized (토큰 만료)
    FE->>BE: 토큰 재발급 요청<br/>(POST /api/auth/reissue, refreshToken)

    alt 재발급 성공
        BE-->>FE: 새 accessToken / refreshToken
        FE->>BE: 원래 요청 재시도 (새 토큰)
        BE-->>FE: 정상 응답
        FE-->>User: 끊김 없이 기능 수행
    else 재발급 실패
        BE-->>FE: 재발급 거부
        FE-->>User: 자동 로그아웃 → 로그인 페이지 이동
    end
```

## 3. WBS (Work Breakdown Structure)

프로젝트를 기능 단위로 분해한 작업 구조. (담당자·기간은 팀 상황에 맞게 채워 사용)

### 3-1. 트리 구조

```mermaid
flowchart TD
    P[PetFit 프로젝트]

    P --> A[1. 기획/분석]
    P --> B[2. 설계]
    P --> C[3. 프론트엔드 개발]
    P --> D[4. 백엔드 연동]
    P --> E[5. 테스트/마무리]

    A --> A1[1.1 요구사항 정의]
    A --> A2[1.2 기능 명세]
    A --> A3[1.3 화면 설계/와이어프레임]

    B --> B1[2.1 UI/UX 디자인]
    B --> B2[2.2 DB/API 설계]
    B --> B3[2.3 컴포넌트 구조 설계]

    C --> C1[3.1 공통/레이아웃]
    C --> C2[3.2 회원/인증]
    C --> C3[3.3 상품/쇼핑]
    C --> C4[3.4 AI 스타일링]
    C --> C5[3.5 반려견 프로필]
    C --> C6[3.6 갤러리]
    C --> C7[3.7 마이페이지]

    D --> D1[4.1 인증/토큰 연동]
    D --> D2[4.2 상품/주문 API]
    D --> D3[4.3 AI 스타일링 API]
    D --> D4[4.4 갤러리/리뷰 API]

    E --> E1[5.1 통합 테스트]
    E --> E2[5.2 버그 수정]
    E --> E3[5.3 발표/보고서]
```
